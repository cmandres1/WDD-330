Week 07 Readings

Callbacks:

Many functions are provided by JavaScript host environments that allow you to schedule asynchronous actions. In other words, actions that we initiate now, but they finish later. For instance, one such function is the setTimeout function. The ability of a JavaScript function to accept another function as an argument is a powerful aspect of the language. A caller of the function can pass another function as an argument to execute based on any trigger.

Promise:

Promises in JavaScript are very similar to promises in real life, you make a promise to do something and you will either complete or fail to achieve that promise. There are three possible results for promises:

    Pending: When the promise is still pending (initial state).
    Fulfilled: When the promise is achieved and gets resolved(success).
    Rejected: When the promise is failed(rejected).
We usually use these promises for asynchronous operations (when you have more than one task that needs to be run at the same time). So, we use them to supply the result or the task at some point in the future.

Asynchronous:

Async is a short form for “asynchronous”. Synchronous means executing statements one after the other which implies the next statement will get executed only after the previous statement is executed completely. Whereas in Asynchronous calls the next statement gets executed without even waiting for the previous one’s execution.
